Teknisk oversikt

Teknisk dokumentasjon for Lilletorget drift: driftmiljø, dataflyt, importer, API-er, lokale loggere og hvor feil bør sjekkes først.

Produksjonsmiljø

  • NavnLilletorget drift, tidligere Fibaro10.
  • Offentlig URLhttps://online.lilletorget.net.
  • Lokal apphttp://192.168.20.218:8110 på QNAP.
  • QNAPTS-473A på internt nett. Appdata ligger under /share/CACHEDEV1_DATA/Public/containerdata/fibaro10.
  • Containerfibaro10 kjører FastAPI/uvicorn på port 8110.
  • Proxyfibaro10_proxy kjører Caddy. UniFi forwarder 443 til QNAP port 8443.
  • KodeGitHub-repo lewinordby/fibaro10. Produksjon bygges fra main.

Applikasjon

  • BackendFastAPI, SQLAlchemy async, PostgreSQL, server-renderte Jinja-sider og GZip-komprimering.
  • FrontendJinja-maler med felles CSS i static/app.css, meny/header i static/owner-nav.css og Lilletorget-tema i static/lilletorget-system.css.
  • BrandingLilletorget-logo. Sol/P-symbol åpner/lukker meny, tekstlogo går til Status/Dashboard.
  • Helse/health svarer uten innlogging og viser at appen og datalaget er tilgjengelig.
  • InnloggingBrukernavn og passord. Automatiske loggere bruker egen bruker/API-headere der det er satt opp.
  • RollerMaster, Innstillinger og Vanlig bruker.

Dataflyt

HC3

Poster lys, lux, ventilasjon, temperatur og energi til API-et.

QNAP-loggere

Lokale containere henter SUN2, EasyPark og Roborock.

Manuelle importer

Elvia lastes opp manuelt. Kjøretøyfelt kan fylles via app eller browser-tillegg.

Database

PostgreSQL lagrer egne tabeller for hvert domene og importstatus.

Grensesnitt

Viser status, grafer, tabeller, innstillinger og feilsignaler fra samme grunnlag.

HC3 og lys

  • StyringHC3 Lua-scene styrer alle utelys etter lux, tidsvinduer og config fra API.
  • LoggingLux og lysstatus logges omtrent hvert 5. minutt. Endringer får egen hendelse.
  • TestHC3-testvariabel kan overstyre lux midlertidig. Scenen skal tømme testverdien etter kjøring.
  • Varslerntfy kan varsle når lysgrupper faktisk går av/på.

HC3 og ventilasjon

  • StyringLua-scene vurderer temperatur inne, ute, loft, passiv innluft og estimert belastning.
  • LoggingTemperaturer, viftestatus, modus og begrunnelse logges omtrent hvert 5. minutt.
  • YrYr-data lagres i egen tabell når varselet fornyes. Brukes for visning og fremtidig forhåndslufting.
  • Varslerntfy kan varsle ved vifteendringer.

HC3 og energi

  • RealtimeQuickApps i HC3 sender W omtrent hvert minutt for inntak, varmepumper, belysning, massasje og annet.
  • AkkumulertkWh sendes også, men enkelte målere kan nulle seg. Derfor brukes reset-håndtering i beregninger.
  • DifferanseDifferanse kan leses fra HC3, men appen kan beregne egen differanse fra gruppene.
  • VisningHC3-timesforbruk vises forskjøvet til timen forbruket tilhører, slik at det matcher Elvia bedre.

Parkering og EasyPark

  • KildeEasyPark-data hentes av lokal downloader og importeres i parkering-tabellen.
  • Faste jobberNattkjøring logger ut/inn før henting. I tillegg kjøres faste dagoppdateringer, blant annet kl. 12, 15, 18 og 21.
  • Manuell jobbParkering > Oversikt kan trigge ny henting for nyere periode når brukeren har tilgang.
  • OppdateringImporten oppdaterer eksisterende poster. Pågående parkeringer kan derfor få sluttid og endelig beløp senere.
  • Kjøretøykjoretoy lagrer regnr, interne felt, antall, summer, først/sist sett og område.
  • Nøkkeldatakjoretoy_nokkeldata lagrer merke, modell, årsmodell, farge, klasse og SVV-status når data finnes.

Parkeringsoversikten er operativ: valgt dag, pågående øverst, avsluttede under. Statistikk og områdefordeling ligger på egne undersider.

SUN2

  • DagsfilerGir summer per dag/rom: soltid, antall og omsetning. Brukes for historiske oversikter.
  • EnkeltimerSkrapes løpende, typisk hvert 5. minutt, og brukes i dagslinje og forbruk per seng.
  • DedupliseringEnkeltimer lagres med naturlig nøkkel fra tidspunkt, rom/seng, varighet og brukerdata der tilgjengelig.
  • MedlemmerSynkes periodisk. SUN2-id er stabil nøkkel når navn/visning endres.
  • SengerSengedata gir kobling mellom SUN2-navn, fysisk rom-id, modell og aktiv status.
  • Rom-idFysisk rom-id brukes for å håndtere gamle navneendringer og tidligere rom 10.

Roborock / Renhold

  • Approborock_logger kjører lokalt på QNAP og sender data til hoved-API-et.
  • DataRobotmetadata, status, jobber, planer, forbruksdeler og kart/snapshots der dette er tilgjengelig.
  • FremtidLoggeren er laget som egen app slik at den kan peke til ekstern eller lokal Fibaro10 via én setting.
  • FeilVed gammel status: sjekk Roborock-konto, delt robot, lokal nettverkstilgang og containerlogg.

Elvia og forbruk/seng

  • ElviaJSON-fil lastes opp manuelt under Energi > Elvia. Den brukes som fasit for historisk strøm.
  • ImportImporten er idempotent. Samme fil kan kjøres flere ganger uten å lage duplikater.
  • Forbruk/sengBeregnes fra SUN2 enkelttimer og HC3 differanseforbruk når nøyaktig én seng er aktiv.
  • MålevinduBeregningen venter 2 min etter oppstart, stopper 1 min før slutt, krever minst 3 målinger og krever at andre senger har stoppet minst 3 min.
  • ReberegningKnappen Beregn overskriver gamle beregninger og regner på nytt fra tilgjengelig datagrunnlag.

Nettleserutvidelser

  • Områdebrowser_extensions/parking_name_lookup henter liste over biler uten område og kan skrive område tilbake til API.
  • Navnbrowser_extensions/parking_manual_name brukes for manuell innlegging av navn på samme kjøretøy-API.
  • BatchOmrådeutvidelsen kan hente opptil 1000 regnr fra appen når brukeren har nødvendig tilgang.
  • APIBruker kjøretøyendepunkter for å hente manglende biler og skrive område/navn.

AI

  • FormålAI-søk kan bruke registrerte datasett for frie spørsmål på norsk.
  • NøkkelOpenAI API-nøkkel kan legges inn i AI > Innstillinger eller som miljøvariabel.
  • KvoterChatGPT-abonnement og API-kvote er separate. API krever egen billing/kvote.
  • LoggingAI-spørringer logges i egen tabell for feilsøking og oversikt.

Drift og deploy

  • DeployQNAP oppdateres fra GitHub main og bygges med docker compose -f docker-compose.qnap.yml up -d --build.
  • Composefibaro10 eksponerer 8110. fibaro10_proxy eksponerer 8081/8443 og terminerer HTTPS via Caddy.
  • Caddyonline.lilletorget.net reverse proxies til fibaro10:8110 og setter sikkerhetsheadere.
  • UniFiPort forward fra offentlig 443 til 192.168.20.218:8443. QNAP-admin skal ikke eksponeres eksternt.
  • LoggerSjekk containerlogger for postinger, importer og eventuelle 500-feil.
  • RenderRender er ikke lenger aktivt driftsmiljø og skal ikke brukes for produksjonsflyt.

Feilsøking først

  • 1Åpne Status > Datakilder og se om en kilde er treg, gammel eller feilet.
  • 2Sjekk Status > Dashboard for driftsradar og siste hendelser.
  • 3Sjekk aktuell fagmeny: Parkering, Soling, Lys, Ventilasjon, Energi eller Renhold.
  • 4Ved importfeil: sjekk lokal QNAP-logger/container først, deretter importstatus i appen.
  • 5Ved rare tider: kontroller om kilden leverer lokal tid eller UTC. SUN2, EasyPark og Elvia skal vises som lokal tid, HC3/Yr normaliseres mot Europe/Oslo.
  • 6Ved design etter deploy: sjekk at ny CSS serveres fra QNAP og hard refresh nettleseren hvis gammel cache ligger igjen.
