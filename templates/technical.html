Teknisk oversikt

Kort dokumentasjon for hvordan SUN2 drift er bygget opp, hvor data kommer fra, og hvor feil normalt sjekkes først.

Applikasjon

  • NavnFibaro10 / SUN2 drift
  • Produksjonhttps://fibaro10.onrender.com
  • KodeGitHub-repo deployes til Render fra main.
  • BackendFastAPI med PostgreSQL-database.
  • FrontendServer-renderte HTML-maler, felles CSS og meny.

Tilgang

  • InnloggingBrukernavn og passord/nøkkel lagres som tilgangsnøkler.
  • RollerMaster, Innstillinger og Vanlig bruker.
  • VarselMaster kan abonnere på ntfy-varsel når andre brukere benytter løsningen.
  • AdminBrukere og nøkler ligger under Konto.

Dataflyt

HC3

Poster lys, lux, ventilasjon, temperatur og Yr-data til API-et.

Lokale loggere

QNAP/Docker kan hente Roborock og SUN2-data og sende inn til API-et.

Database

Data lagres i egne tabeller for lys, ventilasjon, renhold, soling, energi og importstatus.

Grensesnitt

Sidene viser status, tidslinjer, logger, grafer og feilsignaler fra samme datagrunnlag.

Hyppige data

  • LysLux og lysstatus logges hvert 5. minutt.
  • VentilasjonTemperaturer, viftestatus og effektgrunnlag logges hvert 5. minutt.
  • YrVærdata hentes når Yr-varselet er utløpt eller må fornyes.
  • RoborockRobotstatus hentes av Roborock_logger og postes til API-et.

SUN2-import

  • DagsfilerSummer per rom og dato.
  • EnkeltimerSolinger med tidspunkt, rom, bruker, varighet og beløp.
  • MedlemmerMedlemsregister importeres separat.
  • SengerRom-/sengmetadata brukes for mer stabile rom-id-er.

Energi

  • ElviaManuell opplasting av timesfil under Energi.
  • FormatImporten lagrer forbruk per time i egen tabell.
  • BrukKan sammenlignes med soling på dagslinje.

Viktige endepunkter

  • Helse/health
  • HC3 inn/events og eldre /log
  • Importstatus/api/import-status/json
  • Lys/lights/json og /lights/samples/json
  • Ventilasjon/ventilation/json og /ventilation/samples/json
  • Soling/api/sun2/sessions/json, /api/sun2/members/json, /api/sun2/beds/json
  • Energi/api/energi/elvia/json

Feilsøking først

  • 1Åpne Status → Datakilder og se om noe er tregt eller gammelt.
  • 2Sjekk Status → Dashboard for Driftsradar og siste hendelser.
  • 3Sjekk aktuell fagmeny: Lys, Ventilasjon, Renhold, Soling eller Energi.
  • 4Ved importfeil: sjekk lokal logger/QNAP først, deretter API-status i appen.
