Lilletorget drift på én side

Appen samler styring og dokumentasjon for bygget: lys, ventilasjon, renhold, soling, energi, datakilder og tilgang. Start alltid på Dashboard når du skal se om alt er som det skal.

Daglig bruk

Åpne DashboardSe nåstatus for inne, ute, lux, vær, lys, ventilasjon og driftsradar.
Les DriftsradarDen viser datakilder, dagens soling, strøm og renhold. “Alt ser rolig ut” betyr at hovedsignalene er friske.
Bruk DagslinjerStatus, Lys, Ventilasjon og Soling har tidslinjer/grafer som viser hva som faktisk har skjedd gjennom dagen.
Gå til fagområdeHvis noe virker feil, åpne riktig hovedmeny og se hendelser, logger og innstillinger der.

Navigasjon

Sol/P-symbolet øverst åpner og lukker menyen. Teksten “Lilletorget” sender deg tilbake til Dashboard. På mobil lukkes menyen automatisk når du velger en side.

Riktig sted å begynne

Status → Dashboard viser det viktigste først. Bruk Status → Datakilder når du vil vite om logging/import faktisk går.

Endringer

Brukere med innstillingstilgang kan endre lys- og ventilasjonsgrenser. Endre én ting om gangen og sjekk loggene etterpå.

Hva menyene brukes til

  • StatusDashboard, samlet dagslinje og datakilder. Dette er kontrollrommet.
  • LysLys-hendelser, lux-logg, dagslogg lux og innstillinger for utelys.
  • VentilasjonHendelser, temperatur-/viftelogging, Yr-logg, temperaturgraf og styringsverdier.
  • RenholdRoborock-oversikt, robotstatus, planlagte jobber, historikk, kart og tekniske robotdata.
  • SolingDagslinje per fysisk rom, statistikk, enkelttimer, senger og medlemmer.
  • EnergiHC3 minuttlogging, Elvia-import og beregnet forbruk per seng.
  • AIFrisøk i datasett når OpenAI-nøkkel og kvote er aktiv. Foreløpig avhengig av ekstern API-kvote.
  • KontoKontooversikt, teknisk dokumentasjon, manual, brukere/tilgang for master og utlogging.

Data og import

Lys

HC3 sender lux og lysstatus hvert 5. minutt. Hendelser logges når lys endrer status. Lux-grafen brukes for å forstå hvorfor lys gikk av eller på.

Ventilasjon

HC3 sender temperaturer, viftestatus, effektgrunnlag og modus hvert 5. minutt. Yr-data lagres sammen med ventilasjonsgrunnlaget når varselet fornyes.

Soling

SUN2 dagsfiler gir dagssummer. SUN2 enkelttimer hentes løpende omtrent hvert 5. minutt, slik at dagslinjen og analyser blir ferskere.

Energi

HC3 logger realtime effekt og akkumulert kWh hvert minutt. Elvia-filen lastes opp manuelt og brukes som fasit over tid.

Renhold

Roborock_logger kjører lokalt på QNAP/Docker og sender robotstatus, planer, jobber, forbruksdeler og kartdata til appen.

Datakilder

Status → Datakilder viser når hver kilde sist var OK. Dette er beste sted å sjekke om en logger har stoppet.

Varsler

På Status kan du abonnere på ntfy-varsler for lys og ventilasjon. Master kan også abonnere på brukervarsler under Konto.

  • Trykk “Abonner varsler” fra mobilen for å åpne ntfy-abonnementet.
  • Lys og ventilasjon varsler når status faktisk endres.
  • Brukervarsler hjelper master å se når andre bruker løsningen.

Tilgangsnivåer

Master administrerer brukere. Innstillinger kan endre styringsverdier. Vanlig bruker kan se status og logger.

  • Brukernavn og passord må oppgis ved innlogging.
  • Master kan se og lage nye nøkler.
  • Alle innlogginger logges under Konto.

Når noe virker feil

Rask feilsøking
  • Start med Status → Dashboard. Se etter “Bør sjekkes” eller gamle tidspunkter.
  • Åpne Status → Datakilder hvis en graf eller tabell virker tom eller gammel.
  • For lys: sjekk Lys → Lux logg og Lys → Hendelser.
  • For ventilasjon: sjekk Ventilasjon → Temp logg, Dagslogg og siste hendelser.
  • For soling: sjekk Soling → Dagslinje og Soling → Enkeltimer. Dagsfil og enkelttimer kan ha ulik oppdateringsrytme.
  • For energi: sjekk Energi → Status for HC3-data og Energi → Elvia for manuell fasitimport.
  • For renhold: sjekk Renhold → Oversikt og aktuell robot før du feilsøker QNAP-loggeren.
